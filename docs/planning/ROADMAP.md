# Roadmap

## Near term

- Align project-level planning with the existing task-level tracker workflow.
- Index the highest-signal architecture, service, database, and API documents.
- Move durable action items into `docs/tasks/TODO.md` and `docs/tasks/BACKLOG.md`.

## Mid term

- Map active feature streams and service work to approved phases.
- Consolidate architecture and contract indexes without duplicating detailed docs.
- Standardize verification expectations across frontend, gateway, services, and tests.

## Long term

- Automate archive and history hygiene where practical.
- Create consistent ADR triggers for cross-cutting changes.
- Tie release and rollback routines back to planning and runbook documents.
