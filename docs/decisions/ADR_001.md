# ADR 001 - Dual Documentation Roots

- Status: Accepted
- Date: 2026-05-13

## Context

The repository needed a mandatory first-read workflow gate and a separate structured location for long-term project planning and memory.

## Decision

Use root `doc/` as the mandatory first-read workflow gate and `docs/` as the structured home for planning, history, architecture, decisions, and tasks.

## Consequences

- Normal work must stop when `doc/` is missing or empty.
- Bootstrap tasks may create or repair both roots once.
- Durable project memory remains in `docs/`.
