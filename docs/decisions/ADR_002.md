# ADR 002 - Two-Level Planning Model

- Status: Accepted
- Date: 2026-05-13

## Context

The repository already used `docs/PHASE-TRACKER.md` and `docs/phase-history/` for task-level execution, but it lacked a durable project-level phase plan.

## Decision

Keep `docs/planning/` as the project-level phase source of truth and keep `docs/PHASE-TRACKER.md` as the task-level tracker until an explicit migration retires it.

## Consequences

- New editing tasks still require an active `docs/PHASE-TRACKER.md`.
- Project-level planning and task-level tracking must stay aligned.
- Completed phase summaries belong in `docs/history/`, while archived task trackers stay in `docs/phase-history/`.
