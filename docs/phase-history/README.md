# Phase Tracker History

This folder stores archived copies of the active phase tracker.

## Intended Workflow

1. Before starting a new implementation task, move the current tracker file to this folder.
2. Use a timestamp plus task-slug filename such as `2026-04-07-153000-phase-tracker-workflow.md`.
3. Create a fresh `docs/PHASE-TRACKER.md` for the new task before editing code.
4. Keep the archived files as task history; do not overwrite them.

## Naming Rules

- Format: `YYYY-MM-DD-HHMMSS-task-slug.md`
- Keep the slug short, lowercase, and hyphen-separated.
- Prefer a task summary such as `workspace-role-fix`, `oppm-export-update`, or `phase-tracker-workflow`.

## Scope

This folder is for archived task trackers and older tracker snapshots.
